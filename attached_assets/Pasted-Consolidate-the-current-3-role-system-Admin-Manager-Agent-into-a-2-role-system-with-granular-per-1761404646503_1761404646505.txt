Consolidate the current 3-role system (Admin, Manager, Agent) into a 2-role system with granular permissions:

NEW ROLE STRUCTURE:

1. ADMIN ROLE (with permission levels):

When inviting/adding an admin, show permission selector with two options:

A) Full Control (Super Admin):
   ✅ All system settings (Shopify, IVR, webhooks)
   ✅ Invite/remove admins and agent
   ✅ Assign custom permissions to other admins
   ✅ View all orders across system
   ✅ Manual order assignment and bulk operations
   ✅ Team management (edit profiles, assign extensions)
   ✅ Leave management (approve/reject)
   ✅ Full analytics and reports access
   ✅ Trigger auto-assignment
   ✅ Access all settings pages

B) Partial Control (Customizable Admin):
   Create a permission checklist (like Meta Business Suite) where the inviting admin can toggle:
   
   Team Management:
   [ ] View team directory
   [ ] Edit agent profiles
   [ ] Assign agent extensions
   [ ] Manage leave requests
   
   Order Management:
   [ ] View all orders
   [ ] Assign orders to agents
   [ ] Bulk assign orders
   [ ] Trigger auto-assignment
   
   Analytics & Reports:
   [ ] View team performance
   [ ] View order analytics
   [ ] Export reports
   
   System Settings:
   [ ] Manage Shopify integration
   [ ] Manage IVR settings
   [ ] Configure webhooks
   
   Save these permissions in the database as a JSON object or separate permission flags.

2. Agent ROLE:
   - View/manage only their assigned orders
   - Make calls to customers
   - Update order status for their orders
   - Manage own attendance/presence
   - Submit leave requests
   - Access team messaging
   - No access to settings or other agents' orders

DATABASE CHANGES NEEDED:

Update the users table:
- Change role field to ENUM: 'admin', 'agent' (remove 'manager')
- Add new column: permissions JSON or TEXT (stores custom permission object for partial control admins)
- Add new column: admin_type ENUM: 'full_control', 'partial_control' (nullable, only for admins)
